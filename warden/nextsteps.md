# Warden Project: Next Steps & Cleanup

This document outlines the architectural improvements, code cleanup, and functional enhancements identified during the codebase analysis.

## 1. Architectural Refactoring

- **De-monolithize `orchestrator.ts`**:
  - Implement a **Strategy Pattern** for scan modes (SAST vs. DAST).
  - Create dedicated `SastWorkflow` and `DastWorkflow` classes to handle mode-specific logic.
  - Reduce the size and complexity of the main `runWarden` entry point.
- **Unify Scanning Logic**:
  - Resolve the code duplication between `src/orchestrator.ts` and `src/agents/watchman/index.ts`.
  - Extract scanning execution and result merging into a shared service layer.

## 2. Code Quality & Robustness

- **Structured Fix Data**:
  - Replace regex-based parsing in `EngineerAgent.applyFix` with structured data from scanners.
  - Ensure scanners output a consistent JSON format for vulnerabilities (e.g., `packageName`, `currentVersion`, `targetVersion`).
- **Abstract Fixer Interface**:
  - Decouple the `EngineerAgent` from npm/Node.js specifics.
  - Introduce an `IFixer` interface and implement an `NpmFixer`.
  - Prepare for future support of other package managers (e.g., `PipFixer`, `CargoFixer`).
- **Centralize Constants**:
  - Move hardcoded strings like `'scan-results'`, `'workspaces'`, and default branch names into `src/constants.ts` or a central configuration object.

## 3. User Experience & Error Handling

- **Unified Progress Reporting**:
  - Create a standard interface for reporting progress (e.g., `ProgressReporter`).
  - Abstract away the use of `ora` spinners so agents don't have direct dependencies on UI libraries.
- **Centralized Shell Execution**:
  - Implement a robust utility for running shell commands with consistent logging, error handling, and timeout management.

## 4. Extensibility

- **Scanner Plugin System**:
  - Refactor `WatchmanAgent` to support dynamically registered scanners.
  - This allows adding new tools (e.g., Trivy, Gitleaks) without modifying the core agent logic.
- **Mock Scanner Abstraction**:
  - Formalize the "demo mode" by creating a `MockScanner` that implements the standard scanner interface for better testing and demonstrations.
